Upgrading

Notes on upgrading breaking changes.

0.4 to Latest

Renamed the extend property from the component definition to prototype to fall more in line with the Web Component spec. It also removes any conflicting similarities with the spec'd extends property and is a better name for what it does.

// 0.4 skate('something', { extend: { ... } }); // 0.5 skate('something', { prototype: { ... } });

0.3 to 0.4

Usage of component names rather than selectors. This greatly increases performance since there's no need to match selectors and removes any ambiguity that selectors can introduce.

// 0.3 skate('[something]', { ... }); // 0.4 skate('something', { type: skate.types.ATTR ... });

Re-implement passing of element parameter to lifecycle callbacks. Stop using this. This is because we don't want to rewrite the context of these callbacks in case they refer to the definition within them using this.

// 0.3 skate('something', { insert: function () { this... } }); // 0.4 skate('something', { insert: function (element) { element... } });

Removed ability to replace element in the ready callback.

skate('something', { ready: function (element, done) { // No longer works. done(newElement); } });

Return an element constructor instead of a Skate instance.

// 0.3 var listener = skate('something', ...); // 0.4 var element = skate('something'); element.listener;

0.2 to 0.3

Removed element parameter from lifecycle callbacks. Use this instead.

// 0.2 skate('something', { insert: function (element) { element... } }); // 0.3 skate('something', { insert: function () { this... } });

0.1 to 0.2

Rename inserted to insert to normalise tense.

// 0.1 skate('something', { inserted: ... }); // 0.2 skate('something', { insert: ... });

Initial to 0.1

Removed each() from the instance so that it doesn't get treated as a way to access elements.

var component = skate('something', lifecycle); // Removed. component.each(doSomethingForEachElement);
